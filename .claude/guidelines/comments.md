# Comment Policy

> **Source**: `.windsurf/rules/comments.md`

## Golden Rule
> **"Code should be self-documenting. Comments explain 'WHY', never 'WHAT' or 'HOW'."**

## ✅ When to Comment
1. **Business Logic Rationale**
2. **Technical Limitations & Workarounds**
3. **Complex Algorithms**
4. **Security & Performance Warnings**
5. **TODO/FIXME** with owner & date

## ❌ Never Comment
- Obvious code (`$user = User::find($id);`)
- Commented-out code (use Git)
- Method name repetition
- Bad naming explanations (fix naming instead)

## Summary
> **"The best comment is the one you don't need to write."**

**Priority**:
1. Clean, self-documenting code
2. Meaningful naming
3. Clear structure
4. Comments for exceptional cases only
